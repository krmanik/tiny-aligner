# TinyAligner Browser App

SvelteKit + Vite web frontend for client-side forced alignment.

---

## Development

```bash
npm install
npm run dev
# → http://localhost:5173
```

Loads `public/model.onnx` and `public/lexicon.json` for inference in the browser.

---

## Production Build

```bash
npm run build
npm run preview
```

Static site in `build/` ready to deploy.

---

## Deployment

See [../docs/browser.md](../docs/browser.md) for Vercel, Netlify, GitHub Pages, or static hosting.

## Model & Lexicon

Two files required in `public/`:

- `model.onnx` — ONNX inference model (generated by `python scripts/export_onnx.py`)
- `lexicon.json` — Phoneme lexicon (auto-generated)

See [../docs/browser.md#model--lexicon](../docs/browser.md#model--lexicon) for details.

---

## Architecture

```
src/lib/alignment/
├── engine.ts      # Main alignment pipeline
├── audio.ts       # Audio decode & Mel-filterbank
├── ctc.ts         # CTC Viterbi alignment
├── lexicon.ts     # Tokenization & phone lookup
├── types.ts       # TypeScript interfaces
├── config.ts      # Constants (frame rate, Mel config)
├── fft.ts         # FFT utilities
└── srt.ts         # SRT subtitle output
```

---

## Configuration

Edit `src/lib/alignment/config.ts` to match training hyperparams (if retraining):

```typescript
export const SAMPLE_RATE = 16000;      // Must match training
export const N_MELS = 40;
export const FFT_SIZE = 512;
export const HOP_LENGTH = 160;
export const OUTPUT_FRAME_S = 0.02;    // 20ms
```

## Troubleshooting

**Model/lexicon 404:**
```bash
cd ..  # repo root
python scripts/export_onnx.py
# Verify browser/public/{model.onnx,lexicon.json} exist
```

**Slow inference:**
Normal for browser (1–2s for 10s audio). Use ≤30s clips for UX.
