# Browser Deployment

Deploy TinyAligner as a fully client-side web app.

---

## Development

```bash
cd browser
npm install
npm run dev
# → http://localhost:5173
```

Loads `public/model.onnx` and `public/lexicon.json` for client-side inference.

---

## Production Build

```bash
cd browser
npm run build
npm run preview
# → Opens at http://localhost:4173 (static site in ./build/)
```

---

## Deployment Platforms

### Vercel (recommended)

```bash
npm install -g vercel
cd browser
vercel
```

Deploys `build/` automatically. ~1 minute setup.

### Netlify

```bash
npm install -g netlify-cli
cd browser
netlify deploy --prod --dir build
```

### GitHub Pages

```bash
npm run build
git add build/
git commit -m "deploy"
git push origin main
# → Enable GitHub Pages from repo settings → ./build/
```

### Static server (local testing)

```bash
python -m http.server 8080 --directory browser/build
# → http://localhost:8080
```

## Model & Lexicon

Two files required in `browser/public/`:

### model.onnx

Generated by:
```bash
python scripts/export_onnx.py
```

- **Input**: `mel` [1, T, 40] (log Mel-filterbank, 16kHz)
- **Output**: `log_probs` [T', 258] (per-frame phone logits)
- **Size**: ~3.7 MB (float32); quantize to INT8 for ~900 KB
- **Frame rate**: 50 Hz (20ms per frame)

### lexicon.json

Auto-generated by `export_onnx.py`. Contains:

```json
{
  "lexicon": {
    "这": ["zh", "e4"],
    "NATASHA": ["N", "AH", "T", "AA", "SH", "AH"]
  },
  "phone2idx": {"<blank>": 0, "AA": 1, ...},
  "idx2phone": {"0": "<blank>", "1": "AA", ...}
}
```

## Features

- **Real-time inference**: Process audio in browser, no server needed
- **Bilingual**: Chinese (pinyin) + English (ARPAbet) phones
- **Outputs**:
  - Word-level timestamps
  - Character-level timestamps (Chinese)
  - Phoneme-level timestamps
  - SRT subtitle format

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Model/lexicon 404 | Run `python scripts/export_onnx.py` in repo root; verify `browser/public/{model.onnx,lexicon.json}` exist |
| WASM binary not found | Check browser console; ONNX Runtime WASM files should be auto-fetched from CDN |
| Slow inference | Browser processing is normal (~1-2s for 10s audio on recent CPUs); use ≤30s clips for UX |

## Architecture

```
browser/src/lib/alignment/
├── engine.ts           # Main pipeline
├── audio.ts            # WAV decode, Mel-filterbank
├── ctc.ts              # CTC Viterbi forced alignment
├── lexicon.ts          # Jieba tokenization, phone lookup
├── types.ts            # TypeScript interfaces
├── config.ts           # Frame rate, Mel config
├── fft.ts              # FFT utilities
└── srt.ts              # SRT format output
```

Key module: `engine.ts::runForcedAlignment()`
- Decodes audio → Mel features
- Runs ONNX model
- Viterbi forced alignment
- Segments phone/word/char timestamps

## Configuration

Edit `browser/src/lib/alignment/config.ts` if you retrain with different hyperparams:

```typescript
export const SAMPLE_RATE = 16000;      // Audio sample rate
export const N_MELS = 40;              // Mel-filterbank bins
export const FFT_SIZE = 512;           // FFT window
export const HOP_LENGTH = 160;         // 10ms frame shift
export const OUTPUT_FRAME_S = 0.02;    // 20ms per model output frame
```

**Must match training config** in `scripts/train.py::dataset.py`.

## Model Size Optimization

Default: 3.7 MB float32 ONNX.

**Reduce via quantization:**
```bash
python scripts/export_onnx.py --quantize int8
# → ~900 KB (10× smaller)
```

(Requires `onnxruntime[quantization]`; accuracy loss <1%)
